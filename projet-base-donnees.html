Base de données qualité de l'air

Ce projet consiste à créer une base de données MySQL pour stocker et analyser des relevés de la qualité de l'air dans différentes régions. L'objectif est d'assurer une gestion efficace des données pour leur consultation et leur analyse.

Fonctionnalités principales :

  • Stockage structuré des données de qualité de l'air avec MySQL.
  • Utilisation de requêtes SQL pour l'analyse des données.
  • Optimisation de la BDD afin d'optimiser le temps pour la consultation des données.
  • Utilisation d'un MCD et d'un MLD pour la modélisation de la base de données.

Modèle Conceptuel de Données (MCD)

Le MCD illustre les entités principales (représentées par les blocs bleus) et leurs relations (représentées par les blocs orange) pour modéliser les informations nécessaires à la gestion des relevés de qualité de l'air. Il permet d'établir la base de notre BDD.

Modèle Logique de Données (MLD)

Après avoir schématisé le MCD, il faut faire le MLD qui traduit le MCD en une structure adaptée à la base de données relationnelle, détaillant les tables et leurs relations. Ensuite, il faut implémenter le MLD dans le logiciel MySQL et la BDD est créée.
